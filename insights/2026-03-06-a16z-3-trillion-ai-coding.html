核心观点

1. Legacy Code Porting 是目前 ROI 最高的企业用例

[11:12]

企业正在大量使用 LLM 进行旧代码迁移，比如从 Cobol/Fortran 迁移到 Java。Google 早期的研究显示，即使是非常平凡的任务（如在大型代码库中替换 Java 库），LLM 也能表现出色。访谈对象表示，从多家企业听到的反馈是：传统迁移流程可以获得约 2 倍的效率提升。这导致一个有趣的现象：企业反而在加速招聘开发者，因为发现了大量"低垂果实"项目——少量前期投入就能节省大量基础设施成本。

2. 开发者行为转变：给 Agent 工具而非人工传递信息

[08:08]

过去开发者需要查阅文档，然后告诉 agent 怎么做。现在，agent 可以直接调用 API 获取所需信息。例如，当 agent 需要实现某个功能时，它可以自己查询最新版本的文档，而不是依赖人类"中间人"传递信息。这种"去掉中间人"的模式正在改变开发工作流，让 agent 更加自主。同时，agent 也需要原生环境来验证代码——在人类 review 之前，先看代码能否运行、UI 是否正常、请求是否正常。

3. Unit Testing 迎来复兴：即使个人项目也开始写测试

[10:17]

过去，个人项目很少写单元测试——毕竟只有一个开发者，知道自己要做什么。但现在，开发者开始为个人项目写单元测试，因为：第一，AI 让写测试变得非常简单；第二，测试能帮助 agent 验证变更是否破坏了其他功能。agent 可能没有完整的项目上下文，而单元测试提供了一种易于消化的验证方式。这是 AI 时代带来的意外收获——代码质量可能反而提升了。

4. 代码审查的抽象层面正在重构

[14:20]

当 AI 生成的代码可能超出人类理解能力时，review 每一行代码变得不切实际。核心问题是：代码还是正确的抽象吗？也许应该转向审查"计划"或"功能描述"。如果 PR 的目的是让其他开发者了解变更，那么两句话的描述加上可测试的环境，可能比逐行审查代码更有价值。现有的 GitHub PR 流程可能需要重新设计——AI 已经可以自动评论、指出安全漏洞、检查实现与规范的一致性，但完全依赖 AI 审查的企业目前还没有出现。

5. LLM 作为新型"编译器"的双向能力

[18:04]

传统编译器将高级抽象转换为低级代码，且能严格验证语法正确性。LLM 也能将高层描述转换为代码，但缺乏编译器的严格验证机制。不过，LLM 有一个独特能力：它可以将手写优化的低级代码反向更新到高层抽象（文档）。这创造了一个双向流动的可能性——不仅能从描述生成代码，还能从代码更新描述。这让文档维护变得更容易，也更适合 agent 快速理解项目结构。

6. Git/GitHub 需要为 Agent 高频操作重新设计

[19:50]

Git 仓库是为人类设计的：低频率的 commit、人类阅读的提交信息、为人类设置的 rate limit。但 agent 可能同时运行 100 个并行任务，探索多条路径后回退到其中一条。这种高频、探索性的操作模式与现有设计冲突。需要新的"repo 抽象"：更实时、支持高频率变更、允许中间状态探索而不必永久保存。Relay 等工具正在尝试解决这个问题，提供适合 agent 的 repo 体验。

7. AI 带来的遗留语言复兴

[13:05]

一个有趣的副作用是：由于 LLM 能够处理几乎所有编程语言（包括 Cobol），我们可能会看到遗留语言的"复兴"。过去没人愿意维护的 Cobol 代码，现在可以用自然语言来编写和维护。这并不意味着遗留代码会永远存在，而是迁移它们的成本大幅降低——企业可以更快地从昂贵的 mainframe 迁移到现代基础设施。

8. 企业级工具需要为 AI-native 重构

[22:20]

如果从零开始为 AI 设计，现有的 Confluence、Jira、GitHub 等工具看起来会很不一样。例如，story tracker 是否应该有函数自动查看代码并更新 stories？文档系统是否应该能自动从代码推断并更新文档？Mintify 等新一代工具正在展示这种 AI-native 的设计思路。这不仅是优化现有工具，而是重新思考整个开发工具链在 AI 时代应该长什么样。

我的思考

这个视频揭示了一个重要的趋势：AI 不仅仅是"写代码更快"，而是在重构整个软件开发的工作流和工具链。几个值得深思的点：

1. 工具链的机会：GitHub、Jira、Confluence 这些"标准配置"可能都是待颠覆的对象。谁能先做出 AI-native 的替代品？

2. 中间层消失：很多"传递信息"的人类工作正在被 agent 的直接 API 调用取代。这影响的不仅是编程，而是所有需要"查资料再传递"的工作。

3. 验证比生成更重要：单元测试、运行环境、自动 review 工具——这些"验证"相关的东西反而变得更有价值。AI 能生成，但谁来验证？

4. 遗留系统迁移加速：这对企业 IT 是个巨大的利好消息，但对 IBM 等依赖 mainframe 服务的公司可能是威胁。
